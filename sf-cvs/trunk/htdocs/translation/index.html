Audacity Translation

Audacity is a free audio editor for MacOS, Linux, and Windows. A group of volunteers is translating Audacity into several languages. If you would like to help, write to audacity-translation@lists.sourceforge.net for more information.

About Localization

The abbreviations I18N and L10N stand for internationalization and localization. The numerals refer to the number of letters in the middle of each word. Internationalization allows a computer program to run in different languages and character sets. Localization adds support for a particular language or locale.

We are localizing both the Audacity help files and the program itself. We use localization software to help translate the strings that appear inside the application, including menus, buttons, and dialog boxes. All of the user visible strings are extracted from the Audacity source code and placed into a "message catalog template" file called audacity.pot. The localization software allows you to translate the strings in this file.

These programs can create and edit PO files:

  • poEdit for Windows and Unix.
  • KBabel for KDE.
  • gtranslator for GNOME.
  • GNU gettext is standard on most Unix systems. It includes a PO mode for the Emacs text editor.
  • I don't know of any PO editors for the Mac, but the author of poEdit is working on a version for MacOS X.

Translation instructions

  • First, you will need to download Audacity 1.1. This an experimental version of Audacity for testing and translation. It is very unstable and not ready for regular use. Do not use Audacity 1.1 on important files.

    Audacity 1.0 does not have support for translated messages. This is a new feature in Audacity 1.1.

  • To start a new translation, download the latest audacity.pot file from the Resources section above. Rename it audacity.po when you begin translating. To contribute to an existing translation, download the audacity.po file for your language.

  • When you have begun translating the PO file, use your translation program to compile a message catalog. This will produce a file called audacity.mo. In the same folder as the Audacity 1.1 executable, create a folder whose name is the two-letter code for your language. Place the MO file into this folder.

  • Now, start the Audacity program. From the File menu, open the Preferences window. Go to the Interface tab and enter your two-letter language code. You will need to close the Audacity application and start it again for this change to take effect. After you do, it should load your message catalog and display your translated strings.

Notes

  • In poEdit, make sure to set the "charset" option correctly. Otherwise, poEdit will not save any translations with non-English characters.

  • The string " window" in the audacity.po file does not refer to a window on the computer screen. In this case it means a mathematical function with a "bell curve" shape, used in signal processing.

  • Strings like "&File" and "New &Audio Track" are menu items. The letter after the & character is used as a keyboard shortcut on Windows and Unix.

  • "Import Audio...\tCtrl+i" is also a menu item. The \t stands for a tab character, which separates the menu text from the keyboard command.

  • You will find many strings like "There were %d buffer underruns, last near %lf seconds." The % signs mark places where additional information will be inserted into the string. For example, this string might become "There were 4 buffer underruns, last near 3.01 seconds." If you change the order of the the % markers in a string, add "1$" after the percent sign that used to be first, "2$" after the percent sign that used to be second, and so on. For example:
    "The buffer underrun near %2$lf seconds was the last of %1$d."
